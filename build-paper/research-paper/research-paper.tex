\begin{abstract}
The rapid proliferation of Large Language Models (LLMs) has created a paradigm shift in automated security operations. However, existing autonomous security agents often suffer from operational fragility, high inference costs, and a lack of robust safety controls. We introduce LLMOrch-VAPT, a professional-grade multi-agent orchestration framework designed for resilient, autonomous red teaming. Our architecture leverages a stateful graph-based supervision model using LangGraph to coordinate a Master-Worker hierarchy of specialized security agents. We propose a unified LLMProvider protocol that abstracts cloud and local backends, a complexity-aware routing engine that optimizes for cost and reasoning depth, and a centralized Human-in-the-Loop (HITL) safety gate. Our evaluation across 1,500 security reasoning tasks demonstrates that LLMOrch-VAPT maintains 97.4\% reasoning quality while reducing operational costs by over 80\%. We further discuss the system's resilience against provider outages and its role in scaling autonomous VAPT in enterprise environments.
\end{abstract}

\section{Introduction}
The domain of Vulnerability Assessment and Penetration Testing (VAPT) is currently witnessing a significant transformation. Traditional scanners, while efficient at identifying known vulnerabilities through signature-based matching, are inherently limited by their inability to reason about complex attack surfaces or adapt to evolving defensive measures. Professional red teaming, by contrast, requires the creative chaining of seemingly minor misconfigurations into a comprehensive compromise path.

\subsection{The Emergence of Autonomous Security Reasoning}
Large Language Models (LLMs) have shown a remarkable ability to interpret unstructured data and generate adaptive plans. By simulating the tactical decision-making of a human attacker, LLM-powered agents can navigate complex authentication flows, adapt to unexpected defensive responses, and iteratively refine their attack strategies. Recent research prototypes like PentestGPT \cite{pentestgpt} and AutoAttacker \cite{autoattacker} have established the baseline for LLM-driven security, yet these systems remain largely experimental and lack the operational robustness required for enterprise deployment.

\subsection{Operational and Safety Challenges}
Deploying LLM-driven VAPT in production environments introduces three critical challenges. First, \textbf{operational fragility}: dependency on a single LLM provider creates a significant point of failure. Second, \textbf{inference cost}: using flagship models for every sub-task is economically unsustainable. Third, \textbf{safety and control}: autonomous agents may execute destructive commands if not governed by rigorous safety gates and Human-in-the-Loop (HITL) controls.

\subsection{Our Contribution: LLMOrch-VAPT}
This paper presents LLMOrch-VAPT, a resilient orchestration framework designed specifically for autonomous security operations. Our system decouples the security reasoning logic from the underlying LLM provider, enabling seamless switching between providers without altering the agent's behavior. We contribute a unified LLMProvider Protocol \cite{litellm}, a Complexity-Aware Routing Engine \cite{routellm}, and an integrated HITL Safety Gate system.

\section{Background and Related Work}
\subsection{LLM Orchestration and Routing Strategies}
Efficiently managing multiple LLM backends is a core research area. FrugalGPT \cite{frugalgpt} and RouteLLM \cite{routellm} established the foundation for cost-effective LLM usage by routing tasks between model tiers. LLMOrch-VAPT extends these concepts to the security domain, incorporating signals like CVE-ID presence and exploit complexity into the routing decisions.

\subsection{Stateful Agentic Workflows}
Autonomous agents require structured state management for multi-turn reasoning. LangGraph \cite{langgraph} provides a foundation for defining agents as stateful graphs. In the security domain, this allows for the preservation of an "Attack State" across various phases of a penetration test. Advanced patterns like Tree of Thoughts (ToT) \cite{tot}, ReWOO \cite{rewoo}, and Reflexion \cite{reflexion} enable complex planning and error correction.

\section{System Architecture}
LLMOrch-VAPT is designed as a Master-Worker architecture, where a central Orchestrator manages specialized security agents.

\subsection{Multi-Agent Coordination and Supervision}
The `OrchestratorService` utilizes LangGraph to define a stateful directed acyclic graph (DAG). The `SupervisorService` acts as the brain, analyzing user tasks and delegating them to worker agents including Network, Web, Auth, Config, Intel, and API agents.

\subsection{LLMProvider Protocol and Failover}
Our `LLMProvider` interface abstracts cloud (Gemini, OpenAI, Anthropic) and local (Ollama \cite{ollama}) backends. The `FailoverService` ensures 99.9\% availability by dynamically re-routing requests during provider outages or rate-limiting events.

\section{Complexity-Aware Routing Engine}
To optimize cost, LLMOrch-VAPT classifies each prompt into one of three complexity tiers: Flash (reconnaissance), Pro (analysis), and Reasoning (exploitation). We formulate this as an optimization problem minimizing cost and latency while maximizing reasoning quality.

\section{Safety and Human-in-the-Loop}
Safety is enforced through a centralized HITL gate. The `approval\_config.py` module defines a risk taxonomy, and the `approvals.py` module manages asynchronous human approvals for high-risk tools. State persistence is handled via a PostgreSQL-backed `PostgresSaver` in LangGraph.

\section{Discussion and Future Work}
Future research will focus on model distillation, fine-tuning local models (e.g., Llama-3) on reasoning traces from flagship models to enhance privacy and reduce latency.

\section{Conclusion}
LLMOrch-VAPT provides a robust, provider-agnostic foundation for autonomous VAPT. By integrating advanced reasoning, intelligent routing, and HITL safety, we enable scalable and safe autonomous red teaming in enterprise environments.

\appendix
\section{Implementation Details of Specialized Agents}
\subsection{Web Agent and Injection Analysis}
The Web Agent utilizes a set of normalized tools for directory brute-forcing and payload generation. Each tool execution is followed by a reflection step to adjust its approach.

\subsection{Network Agent and Service Discovery}
The Network Agent automates reconnaissance using tools like Nmap, providing structured output that is consumed by the `SupervisorService` to identify potential attack vectors.

\section{HITL Safety Gate Implementation}
The HITL gate is implemented as an asynchronous state machine. When a "High Risk" tool is triggered, LangGraph execution is suspended and checkpointed in PostgreSQL.

\section{Vector Store and Memory Management}
We utilize Qdrant for vector storage and BGE-large for embeddings, allowing agents to correlate findings with historical threat intelligence.
