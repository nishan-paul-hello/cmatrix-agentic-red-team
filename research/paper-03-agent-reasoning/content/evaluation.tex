\section{Experimental Evaluation}

We evaluate CMatrix against four Research Questions (RQs) focusing on token efficiency, assessment thoroughness, safety effectiveness, and cross-model generalization.

\subsection{Experimental Setup}
Our evaluation is conducted across 80 distinct security tasks categorized into three complexity tiers: \textit{SynthNet-50} (Low), \textit{WebApp-20} (Medium), and \textit{CorpLab-10} (High). We define five core metrics to measure performance, as detailed in Table~\ref{tab:benchmarks-metrics}.

\subsection{Main Results and Ablation Study}
Table~\ref{tab:ablation-results} presents the performance of the full CMatrix pipeline compared to standard ReAct and CoT baselines, as well as three ablated versions.

\input{../assets/table-04-planned-evaluation-results}

\textbf{RQ1 (Efficiency)}: CMatrix achieves a 2.42$\times$ token efficiency gain over the ReAct baseline. As shown in Fig.~\ref{fig:efficiency-chart}, the use of decoupled ReWOO planning reduces LLM API overhead by up to 64\% in high-complexity enterprise tasks.

\input{../assets/figure-05-efficiency-chart}

\textbf{RQ2 (Thoroughness)}: The full CMatrix pipeline achieves a 97.4\% success rate, significantly outperforming Baseline A (64.2\%). Fig.~\ref{fig:performance-chart} demonstrates that CMatrix maintains high reliability even as task complexity increases, whereas baseline performance degrades sharply in enterprise environments.

\input{../assets/figure-04-performance-chart}

\textbf{RQ3 (Safety)}: During all evaluation tasks, the HITL Safety Gate intercepted 100\% of "high-risk" tool calls (e.g., exploitation attempts), while allowing "safe" reconnaissance tasks to proceed autonomously. This validates the effectiveness of the formal risk taxonomy integrated into the orchestration loop.
