\section{Experimental Evaluation}

In this section, we evaluate CMatrix against four Research Questions (RQs) that examine the system's efficiency, thoroughness, safety, and generalization.

\subsection{Experimental Setup}
We conduct our experiments on a high-performance workstation equipped with dual NVIDIA A100 (80GB) GPUs and 256GB of RAM. The framework is implemented in Python 3.10, using LangGraph for orchestration and Redis for state management. 

\subsubsection{Benchmarks and Scenarios}
Our evaluation is conducted across \textbf{80 representative security scenarios} selected from a larger pool of automated test cases. These are categorized into three complexity tiers: \textit{SynthNet-50} (Low), \textit{WebApp-20} (Medium), and \textit{CorpLab-10} (High). We utilize Metasploitable3 and OWASP Juice Shop as the primary target environments.

\subsubsection{Baselines and Parity}
To ensure an \textbf{apples-to-apples comparison} of reasoning patterns, we compare CMatrix against two standard baseline agents, both utilizing the same frontier \texttt{gpt-4o-2024-05-13} backend and identical tool registries:
\begin{itemize}
    \item \textbf{Baseline A (ReAct)}: A standard LangChain ReAct agent.
    \item \textbf{Baseline B (CoT)}: A Chain-of-Thought prompting agent without stateful planning or reflection.
\end{itemize}

\subsubsection{RQ1: Token Efficiency}
CMatrix achieves a 2.42$\times$ token efficiency gain over the ReAct baseline (Fig.~\ref{fig:efficiency-chart}). This is primarily due to the ReWOO planner's ability to plan $M$ steps ahead, avoiding the redundant "summarize-and-prompt" cycle of interleaved loops.

\input{../assets/figure-05-efficiency-chart}

\subsubsection{RQ2: Assessment Thoroughness}
The full CMatrix pipeline achieves a 97.4\% success rate. Fig.~\ref{fig:performance-chart} shows that CMatrix maintains a success rate of 88.4\% in high-complexity enterprise tasks, whereas Baseline A degrades to 32.5\%, validating the necessity of the Reflexion self-correction loop.

\input{../assets/figure-04-performance-chart}

\subsubsection{RQ3: Safety and Operational Latency}
The HITL Safety Gate successfully intercepted 100\% of high-risk tool calls. We distinguish between \textbf{System Latency} (avg. 4.2s per step) and \textbf{Operational Latency}, which includes human approval time. During our trials, human authorization added an average of 42.5s to the total task duration. Despite this, the system maintained an 92\% autonomous throughput for reconnaissance tasks, proving the viability of governed autonomy.

\subsubsection{RQ4: Model Generalization}
Tested across heterogeneous backends, CMatrix allowed \textbf{Llama-3-70B} (82.1\% SR) to approach the performance of a frontier model using standard loops (84.6\%), demonstrating that advanced reasoning patterns bridge the performance gap for smaller, open-weights models.
